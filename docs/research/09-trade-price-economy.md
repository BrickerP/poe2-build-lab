# Finding 09: Trade and economy pain should be solved by handoff links, not price-check promises

## Research result

Players need help buying upgrades, but exact prices are volatile. The site should not promise fixed prices. It should generate search handoffs and explain what to filter for.

## Evidence

Official Path of Exile website includes PoE2 Trade search and bulk exchange, but the public trade UI requires JavaScript and live account/browser interaction to use effectively.

r/pathofexile2builds explicitly says the subreddit is not for price checking because price checking does not correlate with builds. That means build sites have an opportunity to bridge build requirements to trade filters without becoming a price-check forum.

Reddit gear discussions show budget sensitivity: players ask about beginner/budget builds, stat-stacking costs, and whether expensive attribute/resistance combinations are worth it.

## User pain

Players need:

- "show me the cheapest acceptable upgrade"
- "which affixes matter, which are luxury?"
- "how do I search trade for this slot?"
- "what can I ignore at low budget?"

## Product implication

Add trade handoff modules:

- generate filter recipes, not price claims
- show budget tiers: campaign / early maps / red maps / pinnacle
- include warning that prices change by league/day/platform
- link to official trade with pre-filled criteria where possible
- record last manual price check date if human tested
