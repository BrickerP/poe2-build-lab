# Finding 01: Current site lacks real build / gear / attribute capability

## Research result

The current site is a static article MVP. It has beginner pages and a checklist, but it does not yet encode the mechanical objects that PoE2 players actually need:

- no class/ascendancy data model
- no passive tree path or node IDs
- no skill/support gem IDs
- no gear slots, base types, affix priorities, stat thresholds, or resistance targets
- no attribute requirement calculator
- no budget tier or trade search handoff
- no `.build` export/import workflow
- no patch-diff tracking for build validity

## Evidence

Local repo scan on 2026-06-16 found only static HTML generated by `sitegen.py`:

- `builds/best-beginner-builds.html`
- `guides/beginner-guide.html`
- `guides/classes-explained.html`
- `guides/skill-gems-explained.html`
- `guides/currency-guide.html`
- `tools/beginner-build-checklist.html`

The current content is prose guidance. It does not store machine-readable build definitions.

## User pain implied

Players do not only need "best beginner build" text. They need a guided answer to:

- What exact passive nodes should I take?
- Which skills/supports and at what level?
- What gear slots matter first?
- How much Strength/Dexterity/Intelligence do I need after gear swaps?
- What stats make an item an upgrade?
- Can I import this into the in-game Build Planner?

## Product implication

The next site iteration should add a `builds/*.json` data layer and a build renderer that can output:

1. human-readable guide page
2. gear checklist
3. attribute/resistance table
4. in-game `.build` file
5. patch validity/changelog
